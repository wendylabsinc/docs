# Build Configuration

WendyOS build configuration lives in `build/conf/local.conf`. The file is generated by `bootstrap.sh` from the per-board template at `conf/template/boards/<board-id>/local.conf`, which `require`s shared fragments from `conf/template/include/local/`.

Edit `build/conf/local.conf` after bootstrapping to customise your build. The file is not overwritten by subsequent `bootstrap.sh` runs (except that `WENDYOS_LAYER_TREE` is prepended on every run to keep it current).

`bootstrap.sh` also generates `build/conf/auto.conf` on every run. This file is fully regenerated each time, so re-running `bootstrap.sh` without a flag cleanly disables that flag's effects.

---

## bootstrap.sh Flags

### --debug

Enables debug build flags by writing the following to `build/conf/auto.conf`:

```bitbake
WENDYOS_DEBUG = "1"
WENDYOS_DEBUG_UART = "1"
WENDYOS_SSHD = "1"
```

Re-run `bootstrap.sh` without `--debug` to disable these flags.

### --history

Enables buildhistory tracking by writing the following to `build/conf/auto.conf`:

```bitbake
INHERIT += "buildhistory"
BUILDHISTORY_COMMIT = "1"
BUILDHISTORY_FEATURES = "image package"
```

Re-run `bootstrap.sh` without `--history` to disable buildhistory.

## Core Yocto Variables

### MACHINE

The Yocto machine name. Set automatically by `bootstrap.sh` based on the `BOARD` you pass in. Do not change this manually unless you know the implications — it controls which machine config file (`conf/machine/<machine>.conf`) is loaded.

| `BOARD=` | `MACHINE=` |
|---|---|
| `rpi5-sd` | `raspberrypi5-wendyos` |
| `rpi5-nvme` | `raspberrypi5-nvme-wendyos` |
| `jetson-orin-nano-nvme` | `jetson-orin-nano-devkit-nvme-wendyos` |
| `jetson-orin-nano-sd` | `jetson-orin-nano-devkit-wendyos` |
| `jetson-agx-orin` | `jetson-agx-orin-devkit-nvme-wendyos` |
| `jetson-agx-orin-emmc` | `jetson-agx-orin-devkit-emmc-wendyos` |
| `jetson-agx-thor` | `jetson-agx-thor-devkit-nvme-wendyos` |
| `qemu-arm64` | `qemuarm64-wendyos` |

In the standalone repos:

| Repo | Available machines |
|---|---|
| `meta-wendyos-rpi` | `raspberrypi4-64-edgeos`, `raspberrypi5-edgeos` |
| `meta-wendyos-virtual` | `wendyos-vm-arm64` |

### WENDYOS_LAYER_TREE

The per-series layer tree namespace. Set by `bootstrap.sh` and written to both `build/conf/bblayers.conf` and `build/conf/local.conf` on every bootstrap run. It controls which subdirectory of `repos/` holds the active layer clones and which subdirectory of `sstate-cache/` receives build artefacts.

The default value is `scarthgap`. A board's `repos.overrides` file may set a different value (e.g. `whinlatter`) to point at a different series checkout.

Do not change this value manually — re-run `bootstrap.sh` with the appropriate `BOARD` instead.

### DISTRO

The distro name. Set by the template — do not change.

| Repo / target | `DISTRO` |
|---|---|
| `wendyos` repo | `wendyos` |
| `meta-wendyos-rpi` | `edgeos-rpi` |
| `meta-wendyos-virtual` | `edgeos-vm` |

### IMAGE_FSTYPES

Controls the output image formats. Set per machine in `conf/machine/<machine>.conf`. Override in `local.conf` to add or remove formats.

| Format | Used by | Notes |
|---|---|---|
| `wic` | RPi, QEMU, VM | Raw WIC disk image |
| `wic.bmap` | RPi | Block map for bmaptool — enables sparse flashing |
| `wic.bz2` | RPi (standalone layer) | Compressed WIC image |
| `wic.qcow2` | VM | For UTM / Lima / QEMU on macOS |
| `wic.vmdk` | VM | For VMware Fusion |
| `ext4` | All | Raw root filesystem (useful for debugging) |
| `tegraflash-tar` | Jetson AGX Thor (L4T 38.x) | Complete NVIDIA flash package (compressed tar) |
| `tegraflash` | Jetson (L4T 36.x and earlier) | Complete NVIDIA flash package |
| `mender` | Jetson Orin | Mender OTA artifact |

### DL_DIR / SSTATE_DIR / TMPDIR

```bitbake
DL_DIR    = "${TOPDIR}/../downloads"
SSTATE_DIR = "${TOPDIR}/../sstate-cache/${WENDYOS_LAYER_TREE}"
# TMPDIR = "/home/dev/yocto-tmp"  # set to a Docker volume path on macOS
```

`SSTATE_DIR` is partitioned per layer tree so builds targeting different Yocto series on the same workspace do not share sstate signatures. `DL_DIR` is shared across all trees — source tarballs are keyed by URL and checksum and are always reusable.

On macOS builds inside Docker, `TMPDIR`, `DL_DIR`, and `SSTATE_DIR` are pointed at Docker volume mount paths (e.g. `/home/dev/yocto-tmp`, `/home/dev/downloads`, `/home/dev/sstate-cache`) to ensure case-sensitive storage. The `dockerfile.config` mounts these volumes automatically.

### PACKAGE_CLASSES

All WendyOS distros use `package_rpm` (RPM packages). Changing this is not recommended as it affects the entire sstate cache.

### CONF_VERSION

Must remain `"2"`. Used by BitBake to detect incompatible `local.conf` format changes.

## WendyOS-Specific Variables

These variables are defined in the distro conf (`conf/distro/wendyos.conf`) with defaults that can be overridden in `local.conf` or in a machine conf.

### WENDYOS_MENDER

Master switch for the Mender OTA stack and everything that depends on the `/data` partition Mender provides (`mender-esp`, `systemd-mount-containerd`, `swapfile-setup`, `wendyos-etc-binds`, `wendyos-user-data-setup`, etc.).

```bitbake
WENDYOS_MENDER = "1"   # enable Mender stack and /data partition (default for Orin tegra234)
WENDYOS_MENDER = "0"   # disable Mender (default for QEMU, RPi, and Thor tegra264)
```

The smart default enables Mender for all Tegra machines except tegra264 (Thor), and disables it for `qemuall` and `rpi`. Override per-machine or in `local.conf` to opt in or out independently of the SoC.

### WENDYOS_DEBUG

Enable the `debug-tweaks` image feature, which sets an empty root password, enables `PermitEmptyPasswords`, and enables `PermitRootLogin` via SSH.

```bitbake
WENDYOS_DEBUG = "1"   # enable debug-tweaks (development builds)
WENDYOS_DEBUG = "0"   # production image — no empty passwords, no passwordless root SSH (default)
```

The default (`"0"`) is set with a weak `??=` assignment in `conf/template/include/local/common.inc`, so it can be overridden by `build/conf/auto.conf` (written by `bootstrap.sh --debug`), `local.conf`, or a board config without the default clobbering those values. BitBake parses `auto.conf` before `local.conf`; both override the `??=` default.

### WENDYOS_DEBUG_UART

Enable verbose UART debug output during boot.

```bitbake
WENDYOS_DEBUG_UART = "1"   # verbose boot output
WENDYOS_DEBUG_UART = "0"   # silent (default for production)
```

Like `WENDYOS_DEBUG`, the default is a `??=` weak assignment and can be overridden via `auto.conf` (`bootstrap.sh --debug`), `local.conf`, or a board config.

### WENDYOS_SSHD

Include the OpenSSH server (`sshd`) in the image.

```bitbake
WENDYOS_SSHD = "1"   # include openssh-sshd (ssh-server-openssh image feature)
WENDYOS_SSHD = "0"   # omit sshd from the image (default)
```

Set `WENDYOS_SSHD = "1"` in `local.conf` to add SSH access to a build, or use `bootstrap.sh --debug` to enable it (along with `WENDYOS_DEBUG` and `WENDYOS_DEBUG_UART`) via `auto.conf`. The default is a `??=` weak assignment so it is not clobbered by `auto.conf`.

### WENDYOS_PERSIST_JOURNAL_LOGS

Control whether systemd-journal logs are written to persistent storage or to a tmpfs that is cleared on reboot.

```bitbake
WENDYOS_PERSIST_JOURNAL_LOGS = "1"   # logs survive reboots
WENDYOS_PERSIST_JOURNAL_LOGS = "0"   # volatile logs in tmpfs
```

The default tracks `WENDYOS_MENDER`: when Mender provides a `/data` partition, logs are persisted there; when there is no `/data` partition (QEMU, RPi, Thor Phase 1) logs are volatile. Override per-machine or in `local.conf` to decouple this from the Mender setting.

The mechanism used to apply this setting differs by oe-core series: scarthgap uses `VOLATILE_LOG_DIR`; whinlatter and later use `FILESYSTEM_PERMS_TABLES` selection. WendyOS handles this automatically — use `WENDYOS_PERSIST_JOURNAL_LOGS` and the correct backend is selected for the active series.

### WENDYOS_USB_GADGET

Enable the USB gadget composite device (NCM network + ACM serial) on the `usb0` interface.

```bitbake
WENDYOS_USB_GADGET = "1"   # enabled (default for Jetson and RPi5)
WENDYOS_USB_GADGET = "0"   # disabled
```

When enabled, the machine conf sets `ENABLE_DWC2_PERIPHERAL = "1"` (RPi) or the appropriate Tegra USB gadget configuration.

### WENDYOS_USB_NET_MODE

The network mode for the USB gadget `usb0` interface (only effective when `WENDYOS_USB_GADGET = "1"`).

```bitbake
WENDYOS_USB_NET_MODE = "link-local"   # self-assigned 169.254.x.x, mDNS discovery (default)
WENDYOS_USB_NET_MODE = "dhcp-client"  # device is DHCP client, host must run a DHCP server
WENDYOS_USB_NET_MODE = "dhcp-server"  # device runs NM shared mode with dnsmasq
```

### WENDYOS_MDNS_INTERFACES

The network interface(s) on which Avahi publishes mDNS services.

```bitbake
WENDYOS_MDNS_INTERFACES = ""      # all interfaces — discoverable on LAN (default)
WENDYOS_MDNS_INTERFACES = "usb0"  # USB gadget interface only
```

### WENDYOS_CONFIG_PART_SIZE_MB / WENDYOS_CONFIG_PART_NUMBER

Size and partition number of the FAT32 config partition. This partition is mountable on Linux, macOS, and Windows for easy device configuration without SSH.

```bitbake
WENDYOS_CONFIG_PART_SIZE_MB = "256"   # default: 256 MB
WENDYOS_CONFIG_PART_NUMBER  = "16"    # default: partition 16
```

### WENDYOS_FLASH_IMAGE_SIZE (Tegra only)

Controls the size of the flash image created by `doexternal.sh` or `dosdcard.sh`. Must match the capacity of your target storage device.

```bitbake
WENDYOS_FLASH_IMAGE_SIZE = "4GB"    # ~1.3 GB per Mender rootfs partition
WENDYOS_FLASH_IMAGE_SIZE = "8GB"    # ~2.9 GB per rootfs partition
WENDYOS_FLASH_IMAGE_SIZE = "16GB"   # ~6 GB per rootfs partition
WENDYOS_FLASH_IMAGE_SIZE = "32GB"   # ~12 GB per rootfs partition
WENDYOS_FLASH_IMAGE_SIZE = "64GB"   # ~25 GB per rootfs partition (default)
```

The value set in `local.conf` must exactly match the `-s` argument passed to `doexternal.sh` when creating a flashable image file. Mismatches produce a corrupted or non-bootable image.

### WENDYOS_CONTAINER_RUNTIME

Include the container runtime stack (containerd, runc, nerdctl, CNI).

```bitbake
WENDYOS_CONTAINER_RUNTIME = "1"   # include containerd (default)
WENDYOS_CONTAINER_RUNTIME = "0"   # omit (not typical)
```

### WENDYOS_UPDATE_BOOTLOADER

Allow the build to update bootloader firmware (Tegra only).

```bitbake
WENDYOS_UPDATE_BOOTLOADER = "1"   # update bootloader on flash (default for Orin)
WENDYOS_UPDATE_BOOTLOADER = "0"   # skip bootloader update (RPi, QEMU, and Thor Phase 1)
```

### WENDYOS_EFI_DEBUG

Build UEFI firmware in DEBUG mode (verbose output to UART) instead of RELEASE.

```bitbake
WENDYOS_EFI_DEBUG = "0"   # RELEASE mode — minimal output (default for production)
WENDYOS_EFI_DEBUG = "1"   # DEBUG mode — verbose UEFI boot output
```

## RPi-Specific Variables

These are standard meta-raspberrypi variables used in the WendyOS machine configurations.

| Variable | Default | Description |
|---|---|---|
| `ENABLE_UART` | `1` | Enable UART serial console |
| `ENABLE_I2C` | `1` | Enable I2C interface |
| `ENABLE_SPI_BUS` | `1` | Enable SPI interface |
| `ENABLE_DWC2_PERIPHERAL` | `1` | Enable USB gadget (dwc2 peripheral mode) |
| `GPU_MEM` | `16` | GPU memory split in MB (headless) |
| `DISABLE_VC4GRAPHICS` | `1` (RPi4), `0` (RPi5) | Disable/enable VC4 GPU driver |
| `RPI_USE_U_BOOT` | `1` (RPi4), `0` (RPi5) | Use U-Boot bootloader |
| `RPI_EXTRA_CONFIG` | see machine conf | Extra lines appended to `config.txt` |
| `WKS_FILE` | `wendyos-rpi.wks` | WIC kickstart file for partition layout |
| `LICENSE_FLAGS_ACCEPTED` | `"synaptics-killswitch"` | Accept RPi firmware license |

### EDGEOS_DEBUG / EDGEOS_PERSIST_JOURNAL_LOGS (standalone RPi and VM layers)

The standalone `meta-wendyos-rpi` and `meta-wendyos-virtual` layers use these older variable names instead of the `WENDYOS_*` prefix used by the unified `wendyos` repo:

```bitbake
EDGEOS_DEBUG = "1"
EDGEOS_PERSIST_JOURNAL_LOGS = "1"
```

The `wendyos-vm-arm64` machine defines QEMU launch parameters directly in `conf/machine/wendyos-vm-arm64.conf`:

| Variable | Value | Description |
|---|---|---|

| `KMACHINE` | `qemuarm64` | Kernel BSP definition |
| `KBRANCH` | `v6.6/standard/qemuarm64` | Kernel branch |

## Distro Features

All WendyOS distros start from Poky and make these adjustments:

**Added:**
- `systemd usrmerge` — systemd as PID 1
- `networkmanager` — NetworkManager for network configuration
- `virtualization` — containerd and container tools
- `bluetooth` — BlueZ (RPi full images)
- `opengl` — for TensorRT (Tegra only)
- `polkit` — for rtkit real-time audio priority (Tegra only)

**Removed:**
- `x11 wayland sysvinit ptest 3g` — not needed for headless embedded targets

## Disk / Image Sizes

Default image rootfs allocation (set in image recipes):

```bitbake
IMAGE_ROOTFS_SIZE ?= "8192"   # 8 MB minimum (expanded by content)
IMAGE_ROOTFS_EXTRA_SPACE:append = "${@bb.utils.contains("DISTRO_FEATURES", "systemd", " + 4096", "", d)}"
```

### RPi WIC Partition Layout (`wendyos-rpi.wks`)

| Partition | Mount | Size | Filesystem |
|---|---|---|---|
| 1 | `/boot` | 256 MB | FAT32 (MBR) |
| 2 | `/` | 4 GB | ext4 |
| 3 | `/data` | 2 GB | ext4 (`noatime`) |

### VM WIC Partition Layout (`wendyos-vm.wks`)

| Partition | Mount | Size | Filesystem |
|---|---|---|---|
| 1 | `/boot` | 256 MB | EFI (GPT, grub-efi) |
| 2 | `/` | 8 GB | ext4 |
| 3 | `/data` | 2 GB | ext4 (`noatime`) |

The `/data` partition holds containerd images, wendy-agent data, and persistent identity files. It can be expanded to fill the remaining storage space after first boot.
